# 02. 优先解析结构化数据

DataCenterMap 这类现代前端站，经常把页面需要的数据放在前端状态里。

这个项目里，我优先读 `window.__NEXT_DATA__`。

## 为什么这样做

直接扒 DOM 容易遇到：

- 文案换行。
- 样式层级变化。
- 标签顺序变化。
- 布尔字段被渲染成图标。
- 数值单位混在展示文本里。

结构化数据更适合转成 JSON 和 CSV。

## 建议流程

1. 打开页面。
2. 读取 `window.__NEXT_DATA__`。
3. 找到页面 props。
4. 从 props 里拆 identity、address、location、overview、specs。
5. 把布尔值、容量、服务标签统一格式化。
6. 保留 `raw` 字段，方便后续回查。

## 兜底策略

结构化数据缺字段时，再读 DOM。

建议顺序：

1. 结构化数据。
2. 页面文本。
3. 链接和按钮跳转。
4. 截图或人工复核。
