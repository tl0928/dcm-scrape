# 04. Watchdog 怎么盯长跑

长跑采集最怕静默卡住。

watchdog 的任务很朴素：定期确认分片还在推进。

## 盯什么

- 分片进程是否存在。
- JSONL 行数是否增长。
- live log 是否更新。
- 目标站点是否能打开。
- 是否出现 403、验证页、连接超时。

## 重启原则

只重启卡住的分片。

不要一发现异常就重启全部。

更稳的顺序：

1. 先检查站点连通性。
2. 再看 JSONL 和 log 的更新时间。
3. 确认卡住后，终止对应分片。
4. 用同样参数带 resume 重启。
5. 重启后刷新汇总 CSV。

## 日志建议

每个分片一个 live log。

watchdog 自己也写一个 log。

这样后面复盘时可以知道：

- 哪一片卡住。
- 卡住多久。
- watchdog 有没有重启。
- 重启后有没有继续增长。
