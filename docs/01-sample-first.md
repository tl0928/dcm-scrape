# 01. 先跑样例

这个项目一开始先做 10 条样例。

原因很简单：全量开始前，字段范围必须先确认。

## 样例阶段要确认什么

- 客户到底看基础信息，还是看 Specs 细项。
- 外部官网是否需要。
- 坐标是否需要。
- Overview 标签是否需要。
- Ecosystem 是否需要。
- CSV 字段名是否能直接看懂。
- JSON 是否保留完整原始结构。

## 样例输出建议

样例阶段建议同时给两个文件：

- JSON：保留完整结构，方便后续扩字段。
- CSV：给客户预览，方便快速确认字段。

## 样例通过后再扩全量

样例确认后，再做这些事：

1. 固定字段表。
2. 确定 JSONL 进度格式。
3. 确定分片方式。
4. 确定失败和重试策略。
5. 开始全量长跑。
